# Open Iconic assets in BlazorMapTiles

This directory is a vendored copy of the Open Iconic distribution used by the `BlazorMapTiles` sample. The assets are present so the client-side portion of the hosted Blazor app can resolve icon CSS and font files directly from `wwwroot`.

## How the sample uses these files

- `Client\wwwroot\css\site.css` imports `open-iconic\font\css\open-iconic-bootstrap.min.css`
- `Client\Components\NavMenu.razor` uses `oi oi-home` and `oi oi-map`

The imported stylesheet references fonts by relative path, so the `font\css` and `font\fonts` layout must remain unchanged if the asset package is updated.

## Why the assets are checked in

`BlazorMapTiles` uses the classic hosted Blazor navigation shell generated by older templates. That shell expects local icon assets rather than a package reference. Keeping the Open Iconic files in the repository makes the sample deterministic, portable, and usable without any external asset host.
